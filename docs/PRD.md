# PRD (Product Requirements Document) - 剩食盲盒系統

## 1. 專案概述
### 背景與動機
為解決學區周邊商家每日多樣化且難以預測的剩餘食材問題，同時滿足學生族群對於節省餐費的強烈需求，我們提出「剩食盲盒系統」。本系統結合了「盲盒」的隨機趣味性與剩食去化的環保理念，創造出雙贏的解決方案。

### 目標用戶
- **學生**：希望以極低價格獲得餐點，且願意嘗試新鮮事物、享受拆盲盒驚喜感的大學生。
- **商家**：學區周邊有處理剩餘食材需求，但不想花太多時間精確分類與定價的餐飲業者。

### 核心價值主張
- **對商家**：透過「盲盒」模式，無需精確登錄每一項剩食，能快速打包並以超低優惠價售出，大幅提升去化效率。
- **對學生**：在節省大量餐費開銷的同時，體驗未知帶來的驚喜與樂趣。

---

## 2. 功能需求

以下為系統的主要功能清單與對應的使用者故事（User Story）：

1. **設定盲盒上架（商家）**
   - 作為一名 **商家**，我希望 **能設定「盲盒」模式（不指定具體菜色，僅標示類別如：便當盲盒、麵包盲盒）並設定超低優惠價**，以便 **快速處理多樣化剩餘食材，提升去化庫存的效率並增加趣味性**。

2. **預約與領取盲盒（學生）**
   - 作為一名 **學生**，我希望 **能以極低價格預約盲盒並在領取時揭曉內容**，以便 **在節省餐費開銷的同時，體驗拆驚喜包的樂趣**。

3. **管理盲盒庫存與時間（商家）**
   - 作為一名 **商家**，我希望 **能隨時調整盲盒的剩餘數量與可領取時間**，以便 **根據當日實際營運與剩食狀況靈活控制釋出量**。

4. **查看歷史訂單與取餐憑證（學生）**
   - 作為一名 **學生**，我希望 **能在個人頁面查看預約紀錄與專屬取餐憑證（如 QR Code 或訂單編號）**，以便 **到店順利核銷並領取盲盒**。

5. **核銷訂單狀態（商家）**
   - 作為一名 **商家**，我希望 **能透過系統快速核銷學生的取餐憑證並更改訂單狀態為「已領取」**，以便 **準確追蹤實際被領走的盲盒數量並避免重複領取**。

## 3. 非功能需求

### 技術限制
- **後端框架**：使用 Python 的 Flask 框架進行開發。
- **前端渲染**：使用 Jinja2 模板引擎處理畫面渲染。
- **資料庫**：使用 SQLite 作為關聯式資料庫，方便初期開發與輕量部署。

### 效能與安全考量
- **效能考量**：在商品列表與訂單頁面實作分頁功能或延遲加載，確保在大量資料下頁面依然能快速載入。
- **安全考量**：
  - 防範 CSRF 攻擊，所有表單提交需進行驗證。
  - 對使用者輸入的內容進行防護與過濾（防範 XSS 與 SQL Injection），確保系統安全。
  - 嚴格落實權限控管：學生僅能預約盲盒與查看個人訂單，商家僅能管理自己的盲盒與核銷自己的訂單。

## 4. MVP 範圍 (Minimum Viable Product)

| 層級 | 功能描述 |
| --- | --- |
| **Must Have (必須有)** | 1. 商家新增、修改與下架盲盒商品（含類別與價格）<br>2. 學生瀏覽可用的盲盒列表並進行預約<br>3. 商家核銷訂單功能<br>4. 學生與商家的基本註冊/登入功能 |
| **Should Have (應該有)** | 1. 預約成功後產生簡單的取餐憑證（如訂單編號）<br>2. 盲盒剩餘數量的即時更新及防超賣機制 |
| **Nice to Have (可以有)** | 1. 盲盒開箱評價與評分系統<br>2. 學生節省金額與剩食減少量的數據統計（環保成就） |
