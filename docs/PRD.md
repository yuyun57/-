# PRD (Product Requirements Document) - 剩食商品上架與搜尋系統

## 1. 專案概述
### 背景與動機
為了幫助學區周邊的小型商家有效管理剩食資源，減少食物浪費，我們建立一個「剩食商品上架與瀏覽搜尋」系統。該系統允許商家將當日剩餘的餐點資訊快速上架至平台，同時提供大學生等消費者友善的搜尋頁面，讓他們能依照金額、商品類型或商家搜尋並以優惠價格購買，達成剩食再利用的環保目的。

### 目標用戶
- 學區周邊小型商家（主要操作者）
- 大學生（終端資訊接收者與潛在消費者）

### 核心價值主張
- **對商家**：提供一個輕量、易用的後台管理介面，能夠快速新增、修改、刪除剩食商品，有效掌握剩食庫存。
- **對學生**：透過系統上架的精確資訊（包含優惠價、剩餘數量、領取時間），方便快速找到物美價廉的餐點。

## 2. 功能需求

以下為系統的主要功能清單與對應的使用者故事（User Story）：

### 1. 新增剩食商品資訊
- 作為一名 **商家**，我希望 **能新增剩食商品資訊（包含商品名稱、原價、優惠價、數量、圖片及領取時間）**，以便 **將今日剩餘的餐點發布到平台上**。

### 2. 商家修改商品資訊
- 作為一名 **商家**，我希望 **能夠隨時修改已上架的商品資訊（如調整價格、修改領取時間或更新圖片）**，以便 **確保平台上顯示的資訊是最正確的**。

### 3. 刪除剩食商品資訊
- 作為一名 **商家**，我希望 **能夠刪除不再需要或錯誤上架的商品資訊**，以便 **維持商品清單的整潔，並避免學生撲空**。

### 4. 商品列表與瀏覽管理
- 作為一名 **商家**，我希望 **能在一個列表頁面上瀏覽所有我新增的商品及其目前狀態與數量**，以便 **快速盤點與管理今日的剩食狀況**。

### 5. 庫存與狀態即時更新
- 作為一名 **商家**，我希望 **能快速將商品狀態標記為「已售完」或「下架」**，以便 **在商品售罄時即時停止向學生展示，避免爭議**。
- 作為一名 **學生**，我希望 **能看到最新且包含完整資訊（優惠價、圖片、可領取時間）的商品展示**，以便 **決定是否前往購買**。

### 6. 商品搜尋與篩選 (新增)
- 作為一名 **學生**，我希望 **能夠依照價格區間或價格高低過濾商品**，以便 **找到符合我預算的食物**。
- 作為一名 **學生**，我希望 **能夠透過商品類型（如便當、麵包等）進行篩選**，以便 **快速看到想吃的餐點**。
- 作為一名 **學生**，我希望 **能夠搜尋特定商家**，以便 **瀏覽我偏好商店的便宜餐點**。

## 3. 非功能需求

### 技術限制
- **後端框架**：使用 Python 的 Flask 框架進行開發。
- **前端渲染**：使用 Jinja2 模板引擎處理畫面渲染。
- **資料庫**：使用 SQLite 作為關聯式資料庫，方便初期開發與輕量部署。

### 效能與安全考量
- **圖片上傳限制**：商品圖片上傳需限制檔案大小（例如 2MB 以下）與格式（僅限 JPG, PNG），以節省伺服器空間並提升載入速度。
- **安全**：
  - 確保表單送出防範 CSRF 攻擊。
  - 對使用者輸入的內容進行過濾與編碼（防範 XSS 攻擊，Jinja2 預設開啟 Autoescape）。
  - 對於修改或刪除商品的操作，需確保只有該商家本人擁有權限。

## 4. MVP 範圍 (Minimum Viable Product)

| 層級 | 功能描述 |
| --- | --- |
| **Must Have (必須有)** | 1. 商家：新增、修改、刪除商品資訊<br>2. 商家：商品列表檢視<br>3. 學生：商品列表瀏覽頁面<br>4. 學生：基本的價格、類型、商家篩選與搜尋功能 |
| **Should Have (應該有)** | 1. 商家：圖片上傳與預覽、簡易狀態切換<br>2. 學生：多條件複合篩選功能、關鍵字模糊搜尋 |
| **Nice to Have (可以有)** | 1. 商家：批次新增或修改、歷史紀錄<br>2. 學生：依照距離排序、收藏商品功能 |
