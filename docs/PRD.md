# PRD (Product Requirements Document) - 剩食買賣系統

## 1. 專案概述
### 背景與動機
為了解決食物浪費的問題，同時幫助學生節省日常餐飲開銷，我們希望建立一個透明、即時的軟體平台。此系統將成為學區周邊小型商家與大學生之間的橋樑，讓商家能將剩餘資源（如當日未售完但品質良好的餐點）轉化為收益，學生則能以更低廉的價格獲取高品質餐點，最終達成環保減碳與節省支出的雙贏局面。

### 目標用戶
- 大學生
- 學區周邊小型商家（如便當店、麵包店、飲料店等）

### 核心價值主張
- **對學生**：以低廉價格買到高品質的一餐，節省生活費。
- **對商家**：減少食物報廢損失，將原本可能丟棄的剩食轉化為實質收入，並提升環保形象。
- **對環境**：減少食物浪費，達到環保減碳的社會責任。

## 2. 功能需求

以下為系統的主要功能清單與對應的使用者故事（User Story）：

### 1. 會員登入系統
- 作為一名 **使用者（學生/商家）**，我希望 **能夠註冊並登入系統**，以便 **安全地管理我的個人資料與後續操作**。
- 作為一名 **系統管理員**，我希望 **能區分學生帳號與商家帳號**，以便 **提供不同的介面與權限**。

### 2. 商品上架系統
- 作為一名 **商家**，我希望 **能夠快速上架今日的剩食或盲盒商品（包含圖片、原價、折扣價與數量）**，以便 **讓學生知道目前有哪些便宜餐點可購買**。
- 作為一名 **商家**，我希望 **能夠隨時編輯或下架商品**，以便 **根據實際庫存情況進行調整**。

### 3. 商品瀏覽搜尋系統
- 作為一名 **學生**，我希望 **能夠瀏覽目前所有上架的剩食商品與盲盒**，以便 **挑選我想吃的餐點**。
- 作為一名 **學生**，我希望 **能依據餐廳名稱、餐點類型或價格進行搜尋與篩選**，以便 **快速找到符合需求的商品**。

### 4. 購物車與下單
- 作為一名 **學生**，我希望 **能將想買的商品加入購物車並結帳**，以便 **確保我能保留這份餐點**。
- 作為一名 **學生**，我希望 **在下單時能選擇取餐時間**，以便 **配合我的行程前往領取**。

### 5. 訂單管理
- 作為一名 **學生**，我希望 **能查看我的歷史訂單與當前訂單狀態（如待取餐、已完成、已取消）**，以便 **追蹤我的購買紀錄**。
- 作為一名 **商家**，我希望 **能看到所有已成立的訂單與取餐狀態**，以便 **準確備餐並核銷已取餐的訂單**。

### 6. 剩食盲盒系統
- 作為一名 **商家**，我希望 **能將零碎、不固定品項的剩食組合成「盲盒」上架**，以便 **更彈性地處理不同種類的剩餘食材**。
- 作為一名 **學生**，我希望 **能以極低的價格購買「盲盒」**，以便 **獲得驚喜感並省下更多餐費**。

## 3. 非功能需求

### 技術限制
- **後端框架**：使用 Python 的 Flask 框架進行開發。
- **前端渲染**：使用 Jinja2 模板引擎處理畫面渲染。
- **資料庫**：使用 SQLite 作為關聯式資料庫，方便初期開發與輕量部署。

### 效能與安全考量
- **效能**：系統首頁加載速度應低於 3 秒，確保使用者有良好的瀏覽體驗；尖峰時段（如用餐後一小時）需能承受基本的併發連線。
- **安全**：
  - 使用者密碼需進行雜湊（Hash）加密儲存（如 bcrypt）。
  - 防範常見的 Web 攻擊，如 SQL Injection 與 XSS（Jinja2 預設開啟 Autoescape）。
  - 確保訂單結帳流程的狀態一致性，防止超賣現象發生（庫存扣減需具備交易完整性）。

## 4. MVP 範圍 (Minimum Viable Product)

| 層級 | 功能描述 |
| --- | --- |
| **Must Have (必須有)** | 1. 商家/學生註冊與登入<br>2. 商家上架/編輯商品功能<br>3. 學生瀏覽商品清單<br>4. 學生下單與商家核銷功能（訂單管理） |
| **Should Have (應該有)** | 1. 剩食盲盒功能<br>2. 購物車功能（允許一次結帳多個商品）<br>3. 簡單的搜尋與篩選功能 |
| **Nice to Have (可以有)** | 1. 學生評價與留言系統<br>2. 數據儀表板（商家查看本月省下多少食物與增加的收益）<br>3. 線上金流串接（初期可先實作現場付款） |
